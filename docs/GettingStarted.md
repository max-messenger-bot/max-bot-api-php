# Начало работы

Для лучшего понимания документации рекомендуем ознакомиться с разделом **Терминология** в [Соглашения](Conventions.md).

## Бот или API-клиент?

Для выбора варианта использования вам необходимо знать следующую информацию:

- Для отправки команд на сервер достаточно использовать **API-клиент**.
- Если Вам необходимо обрабатывать сообщения с сервера, лучше сразу использовать **бота**.
- Если у Вас есть **API-клиент**, Вы можете использовать его для создания **бота**.
- Если у Вас есть **бот**, Вы всегда можете получить используемый им **API-клиент**.
- Объект **событие** содержит ссылку на экземпляр **API-клиента**.

Если Вам необходимо только отправлять простые сообщения, Вы можете рассмотреть вариант отправки сообщений
без использования этого пакета, описанный в разделе [Отправка сообщений](SendingMessages.md).

## Основные идеи, концепты и рекомендации

- Функционал реализован таким образом, что Вы можете использовать его, не читая никакой сторонней документации.
- При использовании параметров функции с несколькими параметрами рекомендуется использовать именованные параметры.
- **Модели запросов** защищены от внешнего вмешательства и в большинстве случаев передача некорректных
  данных в запросе невозможна. Однако Вы можете отправлять на сервер измененные запросы,
  используя специальную [Raw-модель запроса](Extended/RawModel.md).
- **Модели ответов** также защищены от несанкционированного изменения, но Вы можете создать копию объекта ответа
  с измененными данными.
- Вы можете выполнять [собственные запросы](Examples/CustomeRquest.md) к API, используя официальную документацию.
