# Инструменты разработчика

Консольные утилиты для разработки, отладки и управления подписками находятся в папке [`bin`](../bin).

## Запуск

```sh
./bin/max-debug
```

Если прямой запуск недоступен (например, бит исполнения не установлен), запускайте через PHP:

```sh
php bin/max-debug.php
```

## Установка как отдельного проекта

Если запуск инструментов из вашего проекта затруднён (нет доступа к папке `bin`, конфликт зависимостей и т.п.),
установите пакет как отдельный автономный проект:

```sh
composer create-project max-messenger-bot/max-bot-api-php my-bot-tools
cd my-bot-tools
```

После этого все инструменты доступны напрямую из корня проекта:

```sh
./bin/max-debug
./bin/max-subscribe
./bin/polling-to-webhook
```

Или через PHP:

```sh
php bin/max-debug.php
php bin/max-subscribe.php
php bin/polling-to-webhook.php
```

Настройте API ключи один раз в `dev/.keys` — и больше не вводите их при каждом запуске:

```sh
cp dev/.keys.example dev/.keys
```

## Настройка API ключей

Чтобы не вводить API ключ при каждом запуске, создайте файл `dev/.keys`:

```sh
cp dev/.keys.example dev/.keys
```

Формат файла — одна запись на строку:

```
my-bot=ваш-api-токен
test-bot=другой-api-токен
```

При запуске любого инструмента программа предложит выбрать ключ из списка.

## Инструменты

### max-debug

Выводит в режиме реального времени события, поступающие боту через **Long Polling**.

Используется для наблюдения за входящими событиями без написания обработчика.
Если одновременно запущен другой обработчик **Long Polling**, оба будут получать события параллельно.

### max-chats

Выводит список чатов бота с пагинацией (по 5 чатов на страницу).

Показывает для каждого чата: название, идентификатор, тип, статус, количество участников,
время последнего события, описание и ссылку.

### max-delete-chats

Выводит список чатов, где бот является владельцем, и позволяет их удалять.

- Введите номер чата для удаления конкретного чата.
- Введите `555` для удаления всех чатов на текущей странице.
- Нажмите Enter для перехода к следующей странице.

Перед удалением запрашивает подтверждение (`yes`).

### max-subscribe

Регистрирует подписку **Webhook**. Последовательно запрашивает:

1. **API Key** — токен доступа.
2. **URL** — адрес Webhook-endpoint (только `https://`, без порта).
3. **Типы событий** — конкретные типы через запятую или `all` для всех.
4. **Secret** — кодовое слово для заголовка `X-Max-Bot-Api-Secret` (5–256 символов, `A-Za-z0-9_-`).

После сохранения проверяет, что подписка действительно появилась на сервере.

### max-subscribes

Выводит список всех активных подписок **Webhook** с датой создания, версией API и типами событий.

### max-unsubscribe

Выводит список подписок и позволяет удалить выбранную. Перед удалением запрашивает подтверждение (`yes`).

После удаления выводит обновлённый список подписок.

### polling-to-webhook

Получает события методом **Long Polling** и передаёт каждое из них скрипту-обработчику через **php-cgi**,
эмулируя входящий **Webhook**-запрос.

В отличие от прямой обработки в цикле **Long Polling**, скрипт-обработчик запускается заново для каждого события —
поэтому после правки кода обработчика не нужно перезапускать скрипт.

Конфигурация задаётся в файле `dev/.polling-to-webhook.conf`:

```sh
cp dev/.polling-to-webhook.conf.example dev/.polling-to-webhook.conf
```

Параметры конфигурации:

| Параметр  | Описание                                                                                   |
|-----------|--------------------------------------------------------------------------------------------|
| `API_KEY` | API ключ бота для Long Polling (необязателен — если не задан, будет запрошен интерактивно) |
| `SCRIPT`  | Полный путь к скрипту-обработчику (webhook handler)                                        |
| `BIN`     | Путь к интерпретатору `php-cgi`                                                            |
| `SECRET`  | Значение заголовка `X-Max-Bot-Api-Secret`                                                  |
