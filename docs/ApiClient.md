# API-клиент

## Введение

`MaxApiClient` — основной класс для прямого взаимодействия с API Max Messenger Platform. Он предоставляет типизированные
методы для всех поддерживаемых операций API, включая управление чатами, отправку сообщений, работу с участниками и
получение обновлений.

### Ключевые возможности

- **Управление чатами** — создание, редактирование, удаление групповых чатов, управление закреплёнными сообщениями
- **Управление участниками** — добавление и удаление участников, назначение администраторов, управление правами
- **Работа с сообщениями** — отправка, редактирование, удаление сообщений, получение истории сообщений
- **Callback-ответы** — отправка ответов на интерактивные кнопки
- **Обновления** — получение обновлений через long polling, управление WebHook-подписками
- **Информация о боте** — получение и редактирование информации о боте
- **Действия бота** — отправка индикаторов действий (например, "печатает")
- **Загрузка файлов** — получение URL для загрузки медиафайлов, информация о видео

### Инициализация

```php
use MaxMessenger\Bot\MaxApiClient;
use MaxMessenger\Bot\MaxApiConfig;

// Простая инициализация с токеном
$client = new MaxApiClient('your-access-token');

// Или с конфигурацией
$config = new MaxApiConfig(
    accessToken: 'your-access-token',
    baseUrl: 'https://platform-api.max.ru'
);
$client = new MaxApiClient($config);
```

### Особенности использования

- Все методы API имеют строгую типизацию и встроенную валидацию параметров
- Для работы с моделями запросов используются классы из `MaxMessenger\Bot\Models\Requests\`
- Модели ответов возвращаются из `MaxMessenger\Bot\Models\Responses\`
- HTTP-клиент доступен через `$client->getHttpClient()` для выполнения произвольных запросов
- Метод `sendMessage()` автоматически обрабатывает ошибки готовности вложений с повторными попытками

## Методы API

### addMembers

Добавляет участников в групповой чат. Для этого могут потребоваться дополнительные права.

**Параметры:**

| Параметр   | Тип                            | Описание                                      |
|------------|--------------------------------|-----------------------------------------------|
| `$chatId`  | `int`                          | ID чата.                                      |
| `$userIds` | `UserIdsList\|RawModel\|int[]` | Список ID пользователей для добавления в чат. |

**Возвращает:** `ModifyMembersResult` — Результат запроса на изменение списка участников чата.

**Документация API:** [POST /chats/-chatId-/members](https://dev.max.ru/docs-api/methods/POST/chats/-chatId-/members)

---

### answerOnCallback

Отправляет ответ на Callback. Этот метод используется для отправки ответа после того, как пользователь нажал на кнопку.
Ответом может быть обновленное сообщение и/или одноразовое уведомление для пользователя.

**Параметры:**

| Параметр      | Тип                                        | Описание                                                                                                                                                                                                                         |
|---------------|--------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `$callbackId` | `non-empty-string`                         | Идентификатор кнопки, по которой пользователь кликнул (minLength: 1, pattern: `'^\s+$'`). Бот получает идентификатор как часть `Update` с типом `message_callback`. Можно получить из `$update->getCallback()->getCallbackId()`. |
| `$answer`     | `CallbackAnswer\|RawModel\|NewMessageBody` | Ответ на callback: обновленное сообщение и/или уведомление.                                                                                                                                                                      |

**Документация API:** [POST /answers](https://dev.max.ru/docs-api/methods/POST/answers)

### getMessages

Получает сообщения. Возвращает массив сообщений из чата или указанного списка сообщений. Для выполнения запроса нужно
указать один из параметров — `chat_id` или `message_ids`:

- `chat_id` — ID чата для получения сообщений из указанного чата. Сообщения возвращаются в обратном порядке: последние
  сообщения будут первыми в массиве
- `message_ids` — Список ID сообщений (`mid`). Можно указать один идентификатор или несколько

**Параметры:**

| Параметр      | Тип                             | Описание                                                                                                          |
|---------------|---------------------------------|-------------------------------------------------------------------------------------------------------------------|
| `$messageIds` | `array<non-empty-string>\|null` | Список ID сообщений, которые нужно получить (uniqueItems: true). Обязательный параметр, если не указан `$chatId`. |
| `$chatId`     | `int\|null`                     | ID чата, чтобы получить сообщения из определённого чата. Обязательный параметр, если не указан `$messageIds`.     |
| `$from`       | `int\|null`                     | Время начала для запрашиваемых сообщений (в формате Unix timestamp).                                              |
| `$to`         | `int\|null`                     | Время окончания для запрашиваемых сообщений (в формате Unix timestamp).                                           |
| `$count`      | `int<1, 100>`                   | Максимальное количество сообщений в ответе (minimum: 1, maximum: 100). По умолчанию: `50`.                        |

**Возвращает:** `MessageList` — Возвращает список сообщений.

**Документация API:** [GET /messages](https://dev.max.ru/docs-api/methods/GET/messages)

### getUpdates

Получает обновления. Выполняет долгий запрос (long polling). Каждое обновление имеет свой номер последовательности.
Свойство `marker` в ответе указывает на следующее ожидаемое обновление.

Если параметр `marker` **не передан**, бот получит все ранее не полученные обновления.

> Этот метод можно использовать для получения обновлений при разработке и тестировании, если ваш бот не подписан на
> Webhook. Для production-окружения рекомендуем использовать Webhook.

**Параметры:**

| Параметр   | Тип                               | Описание                                                                                                                   |
|------------|-----------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| `$limit`   | `int<1, 1000>`                    | Максимальное количество обновлений для получения (minimum: 1, maximum: 1000). По умолчанию: `100`.                         |
| `$timeout` | `int<0, 90>`                      | Тайм-аут в секундах для долгого опроса (minimum: 0, maximum: 90). По умолчанию: `60`.                                      |
| `$marker`  | `int\|null`                       | Маркер для получения обновлений с конкретной позиции. Для получения всех ранее непрочитанных обновлений, передайте `null`. |
| `$types`   | `array<UpdateType\|string>\|null` | Список типов обновлений, которые бот хочет получить (uniqueItems: true).                                                   |

**Возвращает:** `UpdateList` — Список обновлений.

**Документация API:** [GET /updates](https://dev.max.ru/docs-api/methods/GET/updates)

### subscribe

Подписывает на обновления. Настраивает доставку событий бота через Webhook — основной механизм получения событий в
продуктовых интеграциях. При активной подписке Long Polling не работает.

После вызова метода события отправляются на указанный Webhook-endpoint в виде HTTPS POST-запросов с объектом `Update`.

> Webhook-endpoint должен возвращать **HTTP 200** в течение 30 секунд.\
> Любой другой код ответа или превышение тайм-аута — ошибка доставки.

**Параметры:**

| Параметр        | Тип                                 | Описание            |
|-----------------|-------------------------------------|---------------------|
| `$subscription` | `SubscriptionRequestBody\|RawModel` | Параметры подписки. |

**Документация API:** [POST /subscriptions](https://dev.max.ru/docs-api/methods/POST/subscriptions)

### unsubscribe

Отписывает от обновлений. Отписывает бота от получения обновлений через Webhook. После вызова этого метода бот перестаёт
получать уведомления о новых событиях, и становится доступна доставка уведомлений через API с длительным опросом.

**Параметры:**

| Параметр | Тип                | Описание                                           |
|----------|--------------------|----------------------------------------------------|
| `$url`   | `non-empty-string` | URL, который нужно удалить из подписок на WebHook. |

**Документация API:** [DELETE /subscriptions](https://dev.max.ru/docs-api/methods/DELETE/subscriptions)
