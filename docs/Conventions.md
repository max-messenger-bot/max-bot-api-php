# Соглашения

## Терминология

- **Бот** — это объект класса `\MaxMessenger\Bot\MaxBot`.
- **API-клиент** — это объект класса `\MaxMessenger\Bot\MaxApiClient`.
- **Конфигурация** — это объект класса `\MaxMessenger\Bot\MaxApiConfig`.
- **Обновление** — это объект класса `\MaxMessenger\Bot\Models\Responses\Update` или его дочернего класса.
- **Тип обновления** — Значение, хранящееся в поле `update_type` объекта **обновление**.
- **Событие** — это объект класса, унаследованного от `\MaxMessenger\Bot\MaxBot\Events\BaseEvent`.
- **Модель запроса** — Это класс, унаследованный от `\MaxMessenger\Bot\Models\Requests\BaseRequestModel`.
- **Модель ответа** — Это класс, унаследованный от `\MaxMessenger\Bot\Models\Requests\BaseRequestModel`.

## Соглашения о стиле кода

### Общие правила

- Базовый стандарт: PSR-12
- Все файлы должны начинаться с `<?php` и `declare(strict_types=1);`
- Кодировка файлов: UTF-8
- Конец строки: LF
- Отступы: 4 пробела (2 пробела для yml, yaml)
- Завершающая новая строка: Да
- Удаление пробелов в конце строк: Да
- Максимальная длина строки: 120 символов (если это возможно)

### Именование

- Интерфейсы: Суффикс `Interface`
- Исключения: Суффикс `Exception`

### Пространства имён

- Основной namespace: `MaxMessenger\Bot`
- Автозагрузка PSR-4: `src/` → `MaxMessenger\Bot\`

### Классы

- Используйте `readonly` свойства, когда это возможно
- Статические методы должны вызываться как статические методы, используя ключевое слово static

### Типизация

- Все параметры и возвращаемые значения должны иметь типы
- Используйте `mixed` только когда действительно необходимо
- Используйте PhpDoc для сложных типов и дженериков

### Импорты

- Все импорты должны быть в начале файла после `namespace`
- Используйте `use function` для функций
- Сортируйте импорты алфавитно

### DocBlocks

- Используйте DocBlocks для всех публичных методов
- Описывайте назначение метода, параметры и возвращаемое значение
- Используйте аннотации Psalm: `@psalm-suppress`, `@psalm-var`, `@template`
- Для публичных методов добавляйте тег `@api`
- Краткое содержание (summary) по возможности должно отвечать на вопрос `Что сделать?`

## Структура директорий

```
src/
├── Contracts/                    # Интерфейсы
├── Exceptions/                   # Исключения
│   └── Validation/               # Исключения валидации
├── HttpClient/                   # HTTP-клиент для API-запросов
│   └── Exceptions/               # Исключения HTTP-клиента
│       └── HttpExceptions/       # HTTP-исключения "Max API"
├── MaxBot/                       # Вспомогательные классы бота
│   └── Events/                   # События бота
└── Models/                       # Модели данных "Max API"
    ├── Enums/                    # Перечисления
    ├── Requests/                 # Модели запросов
    └── Responses/                # Модели ответов
```
