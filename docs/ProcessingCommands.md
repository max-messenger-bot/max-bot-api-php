# Обработка команд

## Введение

**Команды** с точки зрения API Max — это обычные сообщения, начинающиеся с символа `/`.

Нет никаких дополнительных ограничений со стороны Max, как должны выглядеть **команды** и что они должны содержать.

**Команду** можно обрабатывать как обычное сообщение. Об обработке сообщений читайте в
разделе [Обработка событий](ProcessingEvents.md).

Для упрощения обработки команд написан отдельный вспомогательный
класс [CommandHandler](../src/MaxBot/CommandHandler.php) и реализован метод `MaxBot::getCommandHandler()`.

**Связанная документация:**

- Примеры [обработки команд](Examples/ProcessingCommands.md)
- Примеры [обработки событий](Examples/ProcessingEvents.md)
- [Обработка событий](ProcessingEvents.md)

## Получение CommandHandler через MaxBot

Вы можете самостоятельно создать объект класса `CommandHandler` и зарегистрировать его как получатель событий «нового
сообщения». Но также Вы можете воспользоваться методом `MaxBot::getCommandHandler()`.

Данный метод создаст экземпляр класса `CommandHandler` и зарегистрирует его первым обработчиком события «новое
сообщение». При повторном запросе будет возвращён тот же самый экземпляр.

```php
use MaxMessenger\Bot\MaxBot;
use MaxMessenger\Bot\MaxBot\Events\MessageCreatedEvent;

$bot = new MaxBot('your-access-token');

// Получение CommandHandler (создаётся автоматически при первом вызове)
$commandHandler = $bot->getCommandHandler();

// Регистрация обработчика конкретной команды
$commandHandler->onCommand('start', function (MessageCreatedEvent $event): bool {
    // Обработка команды /start

    return true; // Отмечаем событие как обработанное
});

// Регистрация обработчика для всех команд
$commandHandler->onCommands(function (MessageCreatedEvent $event): bool {
    // Обработка любой неизвестной команды

    return true; // Отмечаем событие как обработанное
});
```

## Методы CommandHandler

### onCommand

Регистрирует обработчик для конкретной команды.

**Параметры:**

| Параметр   | Тип                                          | Описание                       |
|------------|----------------------------------------------|--------------------------------|
| `$name`    | `non-empty-string`                           | Имя команды (без символа `/`). |
| `$handler` | `Closure(MessageCreatedEvent): (bool\|void)` | Обработчик команды.            |

**Возвращает:** `$this`

### onCommands

Регистрирует обработчик для всех команд (fallback).

**Параметры:**

| Параметр   | Тип                                          | Описание                    |
|------------|----------------------------------------------|-----------------------------|
| `$handler` | `Closure(MessageCreatedEvent): (bool\|void)` | Обработчик для всех команд. |

**Возвращает:** `$this`

### setCommandSeparator

Устанавливает разделитель между командой и дополнительным содержанием.

**Параметры:**

| Параметр            | Тип                      | Описание                                                 |
|---------------------|--------------------------|----------------------------------------------------------|
| `$commandSeparator` | `non-empty-string\|null` | Разделитель команды и данных (например, пробел или `@`). |

**Возвращает:** `$this`

### setCommandMaxLength

Устанавливает максимальную длину имени команды.

**Параметры:**

| Параметр            | Тип            | Описание                                                         |
|---------------------|----------------|------------------------------------------------------------------|
| `$commandMaxLength` | `positive-int` | Максимальная длина команды в символах UTF-8. По умолчанию: `64`. |

**Возвращает:** `$this`

### getCommandSeparator

Возвращает текущий разделитель команд.

**Возвращает:** `non-empty-string\|null`

### getCommandMaxLength

Возвращает текущую максимальную длину команды.

**Возвращает:** `positive-int`

## Особенности обработки команд

### Имя текущей команды и полезная нагрузка

Так как **команда** является частным случаем обычного **события**, нет возможности добавить какое-то отдельное свойство
или метод в объект **события**. Имя обрабатываемой **команды** и полезная нагрузка записываются в `userData` и получить
их можно из **события** следующим способом:

```php
$commandName = $event->userData['__command'];
$payload = $event->userData['__payload'];
```

**Имя команды** по умолчанию ограничено **64** символами в кодировке UTF-8. Команды длиннее этого значения обработаны не
будут. Однако Вы можете изменить это значение, изменив параметр `$commandHandler->commandMaxLength`.

**Полезная нагрузка** (`__payload`) содержит дополнительный текст команды, если установлен разделитель через
`$commandHandler->commandSeparator`. Если разделитель не установлен, `__payload` будет содержать `null`.

### Статус события

При обработке команд важно изменить статус **события** на «обработано», если Вы не хотите, чтобы Ваша **команда** была
обработана другими обработчиками **событий**.

Больше о статусах **событий** можно прочитать в разделе [Обработка событий](ProcessingEvents.md).

```php
$commandHandler->onCommand('start', function (MessageCreatedEvent $event): bool {
    // Обработка команды...

    return true; // Отмечаем событие как обработанное
});
```

### Порядок обработки

Вы можете установить один обработчик на все **команды**, а также установить индивидуальные обработчики на определённую
**команду**.

**Порядок обработки:**

1. В первую очередь запускаются обработчики конкретной **команды** (из `onCommand`)
2. Если **команда** не была обработана, запускаются общие обработчики **команд** (из `onCommands`)

```php
// Сначала будет вызван этот обработчик
$commandHandler->onCommand('help', function (MessageCreatedEvent $event): bool {
    // Специфичная обработка /help

    return true; // Отмечаем событие как обработанное
});

// Этот обработчик вызывается только если команда не обработана
$commandHandler->onCommands(function (MessageCreatedEvent $event): bool {
    $command = $event->userData['__command'];
    // Обработка неизвестных команд

    return true; // Отмечаем событие как обработанное
});
```

### Дополнительное содержание команды

По умолчанию весь текст сообщения считается **командой**, но Вы можете разрешить добавлять **командам** дополнительное
содержание. Для этого установите `$commandSeparator` напрямую или через метод `$commandHandler->setCommandSeparator()`.

```php
$commandHandler->setCommandSeparator(' ');

// Теперь команда "/start arg1 arg2" будет распознана как:
// - $event->userData['__command'] = 'start'
// - $event->userData['__payload'] = 'arg1 arg2'
```

### Ограничения

- Команды обрабатываются только в диалогах (тип чата `ChatType::Dialog`)
- Команда должна начинаться с символа `/`
- Максимальная длина имени команды: 64 символа (настраивается)
- Команды в групповых чатах не обрабатываются автоматически
