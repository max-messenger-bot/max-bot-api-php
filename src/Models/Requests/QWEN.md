# Конвенции разработки

## Модели запросов

- В качестве примера модели можешь брать модели BotCommand, BotPatch, PhotoToken
- Если класс модели запроса имеет конструктор с обязательными атрибутами, то так же должен быть реализован статический метод `new()` с такими же обязательными аргументами
- В методе `new()` для совместимости, все аргументы должны иметь значение по умолчанию, а в коде функции должны быть дополнительные проверки, что в аргументы не передали null в свойства, которые не могут быть null (смотри пример в модели запроса PhotoToken)
- Класс модели должна иметь свойство $data с описанием свойств из схемы, при этом в описание не должны попадать унаследованные свойства
    - Свойства не присутствующие в required должны быть отмечены в описании как опциональные и иметь соответствующие методы с префиксами isset и unset
        - У следующих моделей не должно быть методов с префиксом unset: PhotoAttachmentRequestPayload
    - Свойства отмеченные как deprecated, нужно игнорировать, а так-же их не должно быть в описании $data
    - Тип схемы array должен быть в описании phpdoc как тип list
    - Вложенные типы так же нужно документировать
    - Если схема модели имеет атрибут additionalProperties, используй тип object
- Если свойство схемы имеет атрибут minLength больше 0, то в описании нужно использовать тип non-empty-string
- Если свойство схемы имеет атрибуты minLength, minItems больше 0, то в описании к методам с префиксом new, set и конструкторе в скобках нужно добавить описание как в методе `Button::getText()`
- Если свойство схемы имеет атрибуты maxLength, maxItems, то в описании к методам с префиксом new, set и конструкторе в скобках нужно добавить описание как в методе `Button::getText()`
- В методах с префиксом new, set и конструкторе вместо типов list, list<type>, нужно использовать типи array и type[] соответственно, а в коде при сохранении в $data делать преобразование функцией array_values (смотри пример в модели запроса BotPatch)
    - К другим типам это правило не применять
- В методах с префиксом get, isset, unset генерировать описание в phpdoc не нужно
- Свойство содержащее discriminator должно быть только для чтения
